# hdls-metrics

Daily automated scan of [dlorp](https://github.com/dlorp) repositories. Collects stars, forks, commit frequency, releases, language breakdown, and more.

## Data

- **`repo-data.json`** — canonical metrics, updated daily at 08:00 UTC
- Generated by [GitHub Actions](.github/workflows/daily-scan.yml)

## Usage

### Profile README
The `dlorp/dlorp` profile README is auto-generated from this data by `.github/scripts/generate-profile-readme.py`.

### Vault integration
The knowledge vault's `repo-cards-update` cron reads `repo-data.json` to keep vault cards fresh without shelling into local clones.

## Metrics collected per repo

| Field | Description |
|-------|-------------|
| `stars` | GitHub stars |
| `forks` | Fork count |
| `watchers` | Subscribers |
| `open_issues` | Open issues + PRs |
| `language` | Primary language |
| `languages_top5` | Top 5 languages by bytes |
| `commits_7d` | Commits in last 7 days |
| `latest_release` | Most recent release tag + date |
| `topics` | Repository topics |
| `size_kb` | Repo size in KB |
| `pushed_at` | Last push timestamp |
| `archived` | Archive status |

## Setup

1. Create a Personal Access Token (PAT) with `repo` scope for private repo access
2. Add it as a repository secret named `PAT_TOKEN` in `dlorp/hdls-metrics`
3. Update `.github/workflows/daily-scan.yml` to use `${{ secrets.PAT_TOKEN }}` instead of `${{ secrets.GITHUB_TOKEN }}` (if scanning private repos)

> **Note:** `GITHUB_TOKEN` only has access to the current repo. For cross-repo scanning of private repos, a PAT or GitHub App token is required.
